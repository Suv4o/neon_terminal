Building a Reactive Terminal UI with Nuxt 3 and Tailwind

OCT 27, 2023 | EXEC_TIME: 8ms
[#NuxtJS] [#TypeScript] [#Design]

Nostalgia is a hell of a drug. But when we build tools today, we can't just rely on aesthetics—we need performance, type safety, and component-driven architecture. Here is how I combined the CRT vibes of the 90s with the raw speed of modern web tech.

> Introduction

The goal was simple: create a reading experience that feels like you're jacking into a mainframe, but without the clunky UX of actual legacy systems. We wanted the visual noise of a CRT monitor, the stark contrast of a terminal, but the smooth scrolling and instant routing we expect in 2024.

Using Nuxt 3 provided the server-side rendering necessary for fast initial loads (critical for a blog), while Tailwind CSS allowed us to rapidly prototype the asymmetric layouts and hard borders that define this brutalist aesthetic.

./the_setup.sh

First, we needed to define our design tokens. Instead of soft shadows and rounded corners, we enforce sharp angles and aggressive text glows. The scanline effect is a crucial element, achieved purely through a global CSS overlay.

tailwind.config.ts 
export default defineNuxtConfig({
  theme: {
    extend: {
      boxShadow: {
        'neon-primary': '0 0 10px rgba(238, 95, 83, 0.5)',
      },
      colors: {
        terminal: {
          bg: '#173353',
          surface: '#1b3b60',
          glow: '#ee5f53'
        }
      }
    }
  }
})

Notice how we define specific glow utility classes. Standard drop shadows imply a physical light source casting a shadow onto a surface below. In a terminal UI, the light source is the text itself. Therefore, we use box-shadow and text-shadow to simulate phosphorescent bleed on a dark screen.

> Implementation Details

The typography strategy relies heavily on the juxtaposition of Fira Code for headings and structural elements, and Ubuntu for long-form reading. If we used monospace for the entire article, fatigue would set in quickly.

  • Maintain high contrast ratios for body copy (WCAG AAA if possible).
  • Keep line length under 75 characters to prevent eye tracking loss.
  • Ensure the scanline gradient overlay has pointer-events: none so it doesn't block interactions.
