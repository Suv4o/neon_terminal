# Spec-Kit Context Document
## For: Neon Terminal Blog Redesign

This document provides the full context for building the redesigned trpkovski.com blog using the "Neon Terminal" design system generated by Google Stitch.

---

## Project Overview

We are rebuilding [trpkovski.com](https://www.trpkovski.com) with a new "Neon Terminal" design - a retro-futuristic developer blog aesthetic inspired by 90s terminal interfaces, CRT screen glows, and cyberpunk hardware. The design was generated using Google Stitch's Ideate mode.

**Tech stack:** Nuxt 4 (Vue 3) + Tailwind CSS + TypeScript

**Reference repository:** https://github.com/Suv4o/personal-blog-2023 (the current blog source code - use for understanding site structure, content types, and routing patterns)

**Design reference:** The Stitch-exported screens in `../stitch_neon_terminal_prd/` (HTML + PNG for each page)

---

## Site Structure

The site has 8 pages/views:

1. **Home** (`/`) - Hero with avatar + bio, latest 3 articles, newsletter subscribe form
2. **Articles** (`/articles`) - Paginated blog listing with search/filter, tag-coloured badges
3. **Article Detail** (`/articles/:slug`) - Full blog post with TOC sidebar, code blocks, cover image
4. **The Keyboard Lab** (`/the-keyboard-lab`) - Grid showcase of mechanical keyboards
5. **The Keyboard Lab Detail** (`/the-keyboard-lab/:slug`) - Individual keyboard detail page
6. **Through The Lens** (`/through-the-lens`) - Photography portfolio with 6 category cards
7. **Through The Lens Category** (`/through-the-lens/:category`) - Photos within a category
8. **About Me** (`/about-me`) - Bio, tech stack, career timeline
9. **Get In Touch** (`/get-in-touch`) - Contact form

### Typography

- **Headings:** `Fira Code`, monospace, bold, 24-48px - command prompt style
- **Body:** `Ubuntu`, sans-serif, regular, 18px - Google Fonts, highly legible
- **Labels/Code:** `Fira Code`, monospace, regular, 12-14px - metadata, tags, timestamps
- **Buttons:** `Fira Code`, monospace, semibold, 16px - uppercase, bracketed

### Style Rules

- **Borders:** 2px solid `#f1918b`, 0px border radius (sharp corners everywhere)
- **Shadows:** No drop shadows. Use neon glows: `0 0 12px rgba(238, 95, 83, 0.6)`
- **Cursor:** `crosshair` for interactive terminal elements
- **Transitions:** Under 200ms - must feel "clicky" and instantaneous
- **Scanlines:** Global CRT overlay using `repeating-linear-gradient`
- **Selection:** `background: #ee5f53; color: #173353`

### CSS Custom Properties

```css
:root {
  --color-primary: #ee5f53;
  --color-background: #173353;
  --color-surface: #1b3b60;
  --color-text: #e0e6ed;
  --color-muted: #f1918b;
  --color-accent: #00ff41;
  --font-mono: 'Fira Code', monospace;
  --font-sans: 'Ubuntu', sans-serif;
  --border-width: 2px;
  --glow-primary: 0 0 12px rgba(238, 95, 83, 0.6);
  --scanline-overlay: repeating-linear-gradient(0deg, rgba(0,0,0,0.15), rgba(0,0,0,0.15) 1px, transparent 1px, transparent 2px);
}
```

## Component Patterns

### Terminal Card
Three-part construction used for article cards, keyboard cards, etc:
1. **Hardware Header** - 24px dark bar with `[ ] [ ] [x]` window controls and filename
2. **Media Bay** - Cover image with primary colour overlay, grayscale-to-colour hover
3. **Console Output** - Text content, tags, metadata, action button

### Navigation
- Fixed top bar with backdrop blur
- Site name: `trpkovski.com ðŸ”¥` in Fira Code
- Links in uppercase monospace with coral hover glow
- Mobile: hamburger menu icon

### Action Buttons
- Style: `> READ.EXE`, `> INSPECT.EXE`, `> SEND_TRANSMISSION.EXE`
- Sharp corners, primary border, solid-fill hover with inverted colours
- Neon glow on hover

### Tag Badges
- Format: `[#TagName]`
- Fira Code 12px, coloured background matching the tag colour map
- Used on article cards and article detail pages

### Newsletter / Subscribe
- Terminal prompt style: `> ./subscribe.sh`
- Borderless input with blinking block cursor `â–ˆ`
- `[ENTER]` submit button

### Form Inputs (Get In Touch)
- Labels as terminal commands: `IDENTIFY_USER`, `RETURN_PATH`, `TRANSMISSION_DATA`
- `>` prefix on each input line
- Bottom border only, appears on focus

## Screen-Specific Notes

### Home
- Asymmetric 40/60 split layout (sticky left sidebar with hero, right with article cards)
- Hero: `> whoami` heading with typing animation, cartoon avatar, bio
- 3 latest article cards with terminal frames and cover images
- Newsletter section at bottom: `[ NEWSLETTER_DAEMON ]`

### Articles (Archive)
- Page title: `> /var/log/articles` or `> ls -la ./archive`
- Search filter styled as `> grep "search terms..."`
- Flat list rows: date | title | tags with dotted border separators
- `[ LOAD_MORE.EXE ]` pagination button

### Article Detail
- 3-column layout: TOC sidebar (250px) | content (720px) | whitespace
- Cover image in terminal frame below title and tags
- Reading time as `EXEC_TIME: 4ms`
- TOC updates active section with `>` prefix
- Code blocks with coral left border and `[COPY]` button
- End marker: `EOF` with `[SHARE.SH]` and `> NEXT_POST` buttons

### The Keyboard Lab
- 2-column grid of keyboard cards
- Grayscale-to-colour hover on images
- Specs as terminal readouts: `> Switch: [Topre 45g Silent]`
- Terminal header with hex ID: `ID: 0x2A1F`
- `> INSPECT.EXE` buttons

### Through The Lens
- 3-column gallery (desktop), 2 (tablet), 1 (mobile)
- Portrait-ratio cards (`aspect-[4/5]`)
- Category name overlay with backdrop blur
- Photo count badge: `[ 12 PHOTOS ]`
- Terminal headers: `CAT_01_ASTRO.EXE`

### About Me
- Avatar in terminal frame: `IMG_VIEWER.SH`
- Tech stack as `SYSTEM_CAPABILITIES --LIST` in 3 columns
- Career timeline as `GIT_LOG --CAREER --ONELINE` with commit hashes
- Mobile bottom navigation bar

### Get In Touch
- Terminal-style form: `IDENTIFY_USER`, `RETURN_PATH`, `TRANSMISSION_DATA`
- Submit: `> SEND_TRANSMISSION.EXE`
- Social links as commands: `> ssh github.com/trpkovski`
